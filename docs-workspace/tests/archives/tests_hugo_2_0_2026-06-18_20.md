## 0. Contexte Hugo 2.0 (rappel)

Hugo 2.0 vise un moteur tutoriel multi-postures piloté par état (UIState, mémoire intra-conversation, CTA, isolation multi-tenant). Les campagnes documentées ici **ne prouvent pas** la prod Encoors ni un déploiement RLS prod sans exécution dédiée — elles attestent l’état **local** au moment des dates ci-dessous.

**Doctrine multi-tenant (20/06) :**

- SUPERADMIN = multi-organisation (`X-Organisation-Id` + switcher front).
- ORGADMIN = périmètre organisationnel unique.
- User = une org ; multi-groupes même org ; pas de transfert inter-org.
- Tutor-links (transitoire) = **SUPERADMIN only** (ORGADMIN plus tard).
- RLS = défense complémentaire ; mode strict `HUGO_RLS_STRICT=1` en CI prod-like.
