## 2. Points non modifiés — décision CTO requise

### INC-02 — ACL formateur (priorité décisionnelle)

**État actuel inchangé :**
- `learner_ids_visible_in_group()` classe **TRAINER** dans `is_tutor_like` (avec TUTOR, COORDO).
- Conséquence : `GET /dashboard/groups/{id}/learners/` → `[]` pour `formateur_test_2` même avec 3 `GroupMembership` en DB.
- Le tuteur voit l’apprenant **uniquement** via `TutorLearnerLink`.

**Pourquoi on n’a pas corrigé sans arbitrage :**
- Deux modèles produit incompatibles :
  1. **Formateur = gouvernance pédagogique org/groupe** → voir tous les learners du groupe (comme un orgadmin restreint au groupe).
  2. **Formateur = confidentialité type tuteur** → voir seulement les apprenants explicitement liés (modèle traces / timeline).
- Changer la règle impacte timeline, exports, confidentialité — pas un simple fix serializer.

**Options pour le CTO :**

| Option | Description | Risque | Effort |
|--------|-------------|--------|--------|
| **A (recommandée si formateur = encadrant métier)** | Branche `Role.TRAINER` dans `learner_ids_visible_in_group` : tous les LEARNER du groupe via membership | Moyen — élargit visibilité traces | ~1 j (ACL + tests + doc) |
| **B (statu quo)** | TRAINER reste sur TutorLearnerLink ; UI montre roster memberships (username) sans dashboard learners | Faible | Déjà partiellement fait |
| **C (hybride)** | TRAINER voit liste learners (membership) mais pas timeline verbatim sans lien | Moyen-élevé | ~2–3 j |

**Test de non-régression documenté :** `test_trainer_dashboard_learners_empty_without_tutor_link` — à adapter si option A.
